# 写作规范指南

## 一、数据引用规范

### 1.1 市场规模数据
- **必须具体到数字**："2024年全球MAM市场规模为XX亿美元（来源：XX机构《XX报告2024》）"
- **必须包含增长率**："预计到2030年达到XX亿美元，年复合增长率为XX%（来源：XX机构《XX预测》）"
- **必须说明增长原因**：数字化转型、企业数字营销、AI技术应用、AIGC爆发等

### 1.2 案例引用规范
- **必须多厂商对比**：描述痛点时至少引用3个厂商案例
- **必须覆盖多行业**：案例需覆盖石油/电力/烟草/研究所/博物馆/文化馆/文旅/公安/消防/教育
- **格式要求**：
  ```
  （1）痛点描述。厂商A案例显示...厂商B案例也反映...
  （2）痛点描述。厂商C项目数据显示...厂商D案例表明...
  ```

### 1.3 竞品对比规范
- **必须制作对比表格**：功能、价格、客户类型、典型案例逐项对比
- **必须有具体数字**：不能说"功能更强"，要说"支持XX功能，竞品不支持"
- **必须覆盖3+厂商**：索贝数码、中科大洋、华栖云、米穗科技中选择3个或以上

### 第二章 用户需求分析

#### 2.1 用户画像分析

**2.1.1 企业市场用户画像**
- 石油企业：宣传部门规模、年视频产量、预算范围
- 电力企业：培训部门规模、资料类型、使用场景
- 金融企业：营销部门规模、内容生产频率
- 制造企业：品牌部门规模、素材类型

**2.1.2 政府市场用户画像**
- 博物馆：藏品数量、数字化程度、预算范围
- 文旅局：宣传需求、活动频率、内容类型
- 公安局：执法视音频量、管理需求
- 教育局：教学资源量、共享需求

**2.1.3 媒体行业用户画像**

- 融媒体中心：多平台发布需求、内容生产流程

#### 2.2 核心痛点分析

**写作格式**（必须遵守）：

**【重要】2.2.1 企业市场核心痛点 和 2.2.2 政府市场核心痛点 必须使用汇总表格**

在撰写这两个小节前，必须：
1. 读取 `phase3_enterprise_summary.md` 文件（企业市场汇总表格）
2. 读取 `phase4_government_summary.md` 文件（政府市场汇总表格）
3. 基于汇总表格中的数据进行分析，而非简单罗列原始案例

**企业市场核心痛点写作格式**：
```
企业用户在媒体资产管理中面临以下核心痛点：

（1）品牌素材管理混乱，版本控制困难。根据企业市场汇总表格，快消品牌用户（如阿普塔集团、伊利）普遍面临此问题。阿普塔集团案例显示，采用DAM系统后线索量提升35%；伊利集团案例显示，品牌素材管理效率提升50%。

（2）培训视频分散存储，难以统一管理和复用。制造企业用户（如美的）面临此问题，内容复用率提升3倍。

（3）营销内容生产效率低，无法快速响应市场需求。金融企业和科技企业面临此问题，素材查找时间缩短70%。

（4）跨地域团队协作困难，资源共享受限。零售企业面临此问题，依赖硬盘存储资料导致丢失风险高。
```

**政府市场核心痛点写作格式**：
```
政府/公共机构用户在媒体资产管理中面临以下核心痛点：

（1）档案资料数字化程度低，检索困难。博物馆用户（如故宫博物院）面临此问题，文物高清数字化保护难度大。

（2）声像档案管理不规范，存在丢失风险。公安用户（如余姚公安）面临此问题，13起群众投诉通过调阅视音频数据查清事实。

（3）多部门协作困难，资源共享机制缺失。教委用户（如北京市教委）面临此问题，教育资源共享效率提升。

（4）合规性要求高，审计追溯困难。档案馆用户（如广州市城市建设档案馆）面临此问题，完成280000张照片整理。
```

**汇总表格字段要求**：
| 字段 | 说明 |
|------|------|
| 目标用户类型 | 按行业分类，如：快消品牌、制造企业、金融企业、博物馆、公安等 |
| 用户痛点 | 具体描述用户面临的核心问题 |
| 使用功能 | 用户使用的具体功能模块 |
| 用户收益 | 量化收益，如：效率提升XX%、成本降低XX万 |
| 产品提供商 | 提供解决方案的厂商 |
| 产品价格 | 项目预算或产品价格区间 |

## 三、Word文档格式规范

### 3.1 页面设置
- 纸张：A4
- 页边距：上下2.54cm，左右3.17cm
- 行距：1.5倍

### 3.2 标题格式
- 一级标题：黑体，三号，加粗
- 二级标题：黑体，四号，加粗
- 三级标题：黑体，小四，加粗

### 3.3 正文格式
- 字体：宋体，小四
- 段落：首行缩进2字符

### 3.4 表格格式
- 表头：黑体，五号，加粗，背景色#D9D9D9（浅灰色，RGB: 217,217,217）
- 正文：宋体，五号
- 边框：1磅实线

### 3.5 图表要求
- 必须有图号、图名
- 必须有表号、表名
- **表格表头**：黑体，五号，加粗，背景色#D9D9D9（浅灰色，RGB: 217,217,217）
- 图表清晰美观
