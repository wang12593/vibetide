## Priority Definitions

- **P0 — 阻塞发布**: 核心流程不可用（登录失败、数据丢失、权限绕过）
- **P1 — 核心功能**: 主要功能异常（CRUD 失败、状态流转错误、计算错误）
- **P2 — 一般功能**: 次要功能问题（UI 显示异常、非核心交互失败）
- **P3 — 边界场景**: 极端情况（并发冲突、超大数据、网络中断恢复）

## Test Type Definitions

- **单元测试**: 测试单个函数/方法，mock 所有外部依赖
- **集成测试**: 测试跨层调用链（Action → DAL → DB），使用测试数据库
- **E2E测试**: 模拟用户完整操作流程，通过浏览器自动化
- **手动测试**: 需要人工判断的场景（视觉验证、AI 输出质量）
