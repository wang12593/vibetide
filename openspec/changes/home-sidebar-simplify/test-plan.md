# home-sidebar-simplify 总体测试方案

## 1. 引言

### 1.1 项目背景

VibeTide 首页侧边栏导航重构：将 7 个一级导航精简为 2 个（首页 + 对话），5 个管理/配置类入口（任务、员工、技能、场景、知识）移入"更多"折叠菜单。同时隐藏首页对话框下方的"能力配置"折叠区域，让用户聚焦于对话交互。

### 1.2 测试目标

- 侧边栏导航结构正确：一级入口、更多菜单、管理后台分别按权限正确显示
- 所有路由可达：移入"更多"的入口仍可正常跳转
- 首页布局无回归：隐藏配置区域后无空区域、无布局异常
- 响应式布局正常：侧边栏在展开/收起/移动端三种状态下均正确

## 2. 测试范围

### 2.1 测试内容（范围内）

**功能模块：**

| 模块 | 测试要点 |
|------|---------|
| 侧边栏一级导航 | 只显示「首页」和「对话」两个入口，图标和文字正确 |
| 侧边栏"更多"菜单 | 点击展开后显示 6 个入口：任务、员工、技能、场景、知识、穆兰配置 |
| 侧边栏管理后台 | 只对有 `system:manage_*` 权限的用户显示（关联 permission-system-fix 变更） |
| 侧边栏历史记录 | 展开状态下底部的"历史记录"折叠区域仍正常工作 |
| 侧边栏路由高亮 | 当前页面对应的导航项正确高亮（包括"更多"中的子项） |
| 首页布局 | 首页不再显示"能力配置"折叠区域，无空 div 残留 |
| 首页聊天区域 | 对话功能不受影响，输入框、发送、消息展示正常 |

**非功能特性：**
- 兼容性：Chrome 最新版、Edge 最新版、Safari 最新版、移动端 Chrome/Safari
- 响应式：桌面端（侧边栏展开/收起）、平板端、移动端（侧边栏隐藏）

### 2.2 不测试内容（范围外）

| 排除项 | 原因 |
|--------|------|
| 管理后台页面功能 | 本变更未修改管理后台逻辑 |
| 权限系统细节 | 属于 `permission-system-fix` 变更范围 |
| 首页聊天对话的 AI 逻辑 | 本变更未修改对话引擎 |
| 知识库模块功能 | 属于 `knowledge-base-redesign` 变更范围 |

## 7. 风险分析

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| "更多"菜单入口用户找不到 | 用户困惑 | 中 | "更多"按钮使用明显的 `⋯` 图标 + Tooltip |
| 收起模式下 Popover 定位异常 | 布局错位 | 低 | 在 Chrome/Safari 两个浏览器验证 |
| 首页空 div 残留 | 布局空白 | 已修复 | 12 条规则审查时已移除空 div |
| 关联变更（permission-system-fix）影响侧边栏 | 管理后台对普通用户可见 | 已修复 | `canAccessAdmin` 已移除 `hasAllPerms` 逻辑 |
