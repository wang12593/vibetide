用户询问技能大分类，尤其补充了如系统互操作等之前未覆盖的类型。我会基于技能本质属性与执行逻辑，而非业务场景，明确分类核心原则，避免乱分。
用户提及系统互操作等新技能，我计划按 “技能执行对象和能力本质” 分类，如执行对象为信息、系统 / 工具、人 / 流程的类别。之后，将 42 项技能重新归类并补充新技能，详细说明各分类定义、特征、子分类及示例，重点阐述系统和即时通讯互操作在媒体场景的例子。
我会补充分类依据与原则，告知用户分类原因及扩展新技能方法，即按执行对象归类。因用户为媒体场景，技能示例要贴合实际，如系统互操作提媒体常用系统，即时通讯提常用工具。还要注意与员工配置一致，明确各技能分配员工。最后给出扩展指南，指导用户添加新技能大类及具体技能并分配员工。
AI 技能统一分类框架（媒体专属版）
核心分类原则：按「技能的执行对象 + 能力本质」划分，而非按业务流程划分。这样既能对齐技术实现逻辑（方便接口封装），又能清晰界定员工权限（避免跨界调用），同时完美兼容你提到的系统互操作、即时通讯、网页自动化等新增能力。
共分为7 大一级分类，覆盖媒体全链路所有可能的 AI 能力，每个一级分类下再按功能拆分子类，可无限扩展。

一、7 大技能大类完整定义与媒体专属清单
1. 信息感知类（执行对象：外部 / 内部信息）
核心本质：从各种渠道获取原始信息，不做任何加工和判断，是所有后续能力的基础。
核心特征：输入 = 检索条件 / 数据源，输出 = 原始数据 / 素材，无决策环节。
子类	媒体专属技能示例	适用员工
全网信息检索	全网热点聚合、垂直站点检索（军网 / 国防部 / 新华社）、社交媒体检索、评论抓取	小雷、小策、小子
内部资产检索	历史稿件库检索、媒资系统检索、知识库检索、过往任务记录检索	小子、小文、小策
结构化信息提取	发布会实录提取、PDF 文档解析、表格数据提取、网页正文提取	小子、小审
实时数据监控	关键词预警、热度趋势监控、竞品账号监控、舆情实时监测	小雷

2. 信息加工类（执行对象：已获取的信息）
核心本质：对原始信息进行分析、创作、审核，将数据转化为有价值的内容。
核心特征：输入 = 原始信息 / 素材，输出 = 加工后的内容 / 结论，包含一定的业务判断。
子类	媒体专属技能示例	适用员工
数据分析	新闻价值评分、舆情情感分析、热度预测、用户画像分析、选题成功率统计	小策、小书、穆兰
内容创作	稿件生成、标题生成、导语生成、摘要生成、风格改写、短视频脚本生成	小文、小剪、小发
质量审核	事实交叉验证、敏感词检测、涉密排查、政治导向审核、版权合规审核	小审、顾问、小文
内容结构化	关键词提取、主题分类、标签生成、摘要生成、知识图谱构建	小子、小书

3. 系统互操作类（执行对象：你方现有业务系统）
核心本质：AI 与你已有的内部系统进行数据读写和操作，打通 AI 与现有工作流的最后一公里。
核心特征：输入 = 系统指令 + 数据，输出 = 系统操作结果，需要对接系统 API。
这是你最关心的部分，也是 AI 真正融入现有工作流的关键。
子类	媒体专属技能示例	适用员工
内容管理系统（CMS）操作	新建稿件、写入稿库、更新稿件状态、提交审核、发布上线、撤回稿件	小文、小排、小发
媒资系统操作	上传素材、检索素材、下载素材、打标签、创建素材集	小子、小图、小剪
发布平台操作	绑定账号、发布内容、删除内容、查看发布状态、获取数据统计	小发
办公系统操作	创建任务、更新任务进度、提交审批、下载附件、上传成果	穆兰、所有员工
数据系统操作	读取阅读量数据、导出报表、同步数据到数据库	小书

4. 自动化工具类（执行对象：外部网页 / 本地工具）
核心本质：模拟人类操作外部网页或本地软件，完成重复性的机械动作。
核心特征：输入 = 操作指令 + 目标地址，输出 = 操作结果，不需要系统 API 对接。
子类	媒体专属技能示例	适用员工
网页自动化	页面导航、表单填写、点击按钮、截图存档、下载文件	小子、小审
本地工具自动化	打开 Word、Excel、PS，执行简单操作（如格式调整、批量重命名）	小排、小图
文件处理	格式转换（Word 转 PDF）、压缩解压、批量重命名、合并文件	所有员工
邮件自动化	发送邮件、添加附件、读取邮件、自动回复	穆兰、小发

5. 通讯协作类（执行对象：人 / 团队）
核心本质：AI 作为团队一员，与真人或其他 AI进行沟通和协作。
核心特征：输入 = 消息 / 指令，输出 = 消息 / 协作结果，需要对接即时通讯工具。
子类	媒体专属技能示例	适用员工
即时通讯操作	发送消息、@指定人员、创建群聊、上传文件、撤回消息	穆兰、所有员工
任务通知	任务开始通知、进度更新通知、审核提醒、截止日期提醒	穆兰
协作沟通	向真人提问、收集反馈、确认需求、汇报进度	所有员工
会议辅助	会议纪要生成、待办事项提取、会议提醒	穆兰、小书

6. 流程管理类（执行对象：任务 / 工作流）
核心本质：对整个任务流程进行规划、分解、调度和管控。
核心特征：输入 = 总需求 / 目标，输出 = 任务方案 / 执行计划，包含复杂的决策和调度逻辑。
子类	媒体专属技能示例	适用员工
任务分解	复杂任务拆解、DAG 图生成、依赖关系校验	穆兰
资源调度	员工技能匹配、任务分配、负载均衡	穆兰
进度管控	任务状态跟踪、超时预警、异常处理、冲突解决	穆兰
结果汇总	多员工成果整合、格式统一、质量初检	穆兰
工作流执行	启动预设工作流、暂停 / 恢复工作流、终止工作流	穆兰

7. 多模态处理类（执行对象：非文本内容）
核心本质：处理图片、音频、视频等非文本内容，是未来扩展的重点方向。
核心特征：输入 = 非文本文件 / 素材，输出 = 处理后的非文本内容或文本描述。
子类	媒体专属技能示例	适用员工
图像处理	图片裁剪、尺寸调整、滤镜应用、文字识别（OCR）、图片描述生成	小图
音频处理	语音转文字、文字转语音、音频剪辑、字幕生成	小音
视频处理	视频剪辑、转场添加、字幕合成、封面生成、视频摘要生成	小剪
生成式 AI	图片生成、视频生成、数字人驱动	小图、小剪

二、分类依据与核心逻辑
为什么按这个维度分类？
1.技术实现友好：同一大类的技能可以共用一套技术底座和接口规范。比如系统互操作类都需要对接 API，自动化工具类都需要用 Playwright/Selenium，方便开发和维护。
2.权限管理清晰：可以按大类给员工授权。比如普通员工不能调用系统互操作类的 "发布上线" 技能，只有小发可以；只有穆兰能调用流程管理类的所有技能。
3.扩展能力强：新增任何技能都能找到对应的大类，不需要重构整个分类体系。比如以后要加 AR/VR 处理能力，直接在多模态处理类下新增子类即可。
4.业务理解成本低：每个大类的名称和职责都非常直观，非技术人员也能快速理解。
快速判断一个新技能属于哪个大类的方法
问自己三个问题：
1.这个技能是在获取信息吗？→ 信息感知类
2.这个技能是在加工信息吗？→ 信息加工类
3.这个技能是在操作某个东西吗？
￮操作内部系统 → 系统互操作类
￮操作外部网页 / 工具 → 自动化工具类
￮操作人 / 团队 → 通讯协作类
￮操作任务 / 流程 → 流程管理类
￮操作非文本内容 → 多模态处理类

三、媒体场景技能授权黄金法则
为了避免职责混乱，严格遵循以下授权原则：
1.穆兰（主编）：只能调用流程管理类 + 通讯协作类技能，以及信息加工类的 "新闻价值评分"" 选题优先级排序 "。绝对不能调用内容创作、系统操作等具体执行技能。
2.专业员工：只能调用自己职责相关的 1-2 个大类的技能。比如：
￮小文（记者）：信息感知类（检索）+ 信息加工类（创作 + 自查）+ 系统互操作类（新建稿件）
￮小发（运营）：信息加工类（风格改写）+ 系统互操作类（发布）+ 通讯协作类（通知）
3.通用技能：文件处理、邮件发送等通用自动化工具类技能，所有员工都可以调用。
4.高危技能：发布上线、涉密排查、系统删除等高危技能，只能由特定员工调用，且必须保留真人审核环节。

四、扩展建议
1.优先实现系统互操作类：这是 AI 真正融入你现有工作流的关键。先对接 CMS 和稿库，实现 "AI 写稿→自动写入稿库→提交审核" 的闭环，比单纯的写作能力价值大 10 倍。
2.其次实现通讯协作类：对接企业微信 / 钉钉，让 AI 在你常用的工具里工作，而不是让员工切换到新平台。比如 AI 写完稿后自动在企业微信里 @审核员，审核通过后自动发布。
3.逐步扩展自动化工具类：先实现网页截图、文件格式转换等简单的自动化，再逐步实现更复杂的网页操作。
4.多模态处理类作为未来规划：先把文本能力做扎实，再逐步扩展图片、视频处理能力。
